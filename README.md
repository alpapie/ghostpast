# Ghostpast Frontend

SvelteKit interface that lets users drop files or paste text, then share a short code generated by the Ghostpast backend. It focuses on a frictionless experience with inline loaders, copy helpers, and download management.

## Features
- Drag‑and‑drop uploader with live file list (20 MB per file cap enforced by the API).
- Optional public‑key hint toggle for future encryption workflows.
- Loader/error overlays during share creation and downloads.
- View screen allows copying text, renaming before download, and per‑file progress indicators.

## Requirements
- Node.js 20+
- npm 10+ (ships with recent Node releases)

## Environment
Create `ghostpast/.env` (or `.env.local`) with:

```env
PUBLIC_BACKEND_URL=http://localhost:3000
```

The value must point to the running backend API; SvelteKit injects it at build time via `$env/static/public`.

## Setup
```bash
cd ghostpast
npm install
npm run dev -- --open
```

Useful scripts:

- `npm run dev`: start Vite dev server with hot reloading.
- `npm run build`: create a production build (output in `build/` once an adapter is configured).
- `npm run preview`: preview the production build.
- `npm run check`: run `svelte-check`.
- `npm run lint`: Prettier + ESLint.
- `npm run test`: execute unit tests via Vitest (no suites yet, provided for future use).

## Project Structure
- `src/routes/+page.svelte` — landing form for creating shares.
- `src/routes/view/[code]/` — route that displays shared content and files.
- `src/lib/api.js` — Axios helper used by the load functions.

## Connecting to the Backend
The UI expects the backend to expose the `/api/create`, `/api/data/:code`, and `/api/download/:code/:filename` endpoints. When deploying, ensure `PUBLIC_BACKEND_URL` references the public URL of the backend service (e.g., a Render, Fly.io, or self‑hosted instance).
